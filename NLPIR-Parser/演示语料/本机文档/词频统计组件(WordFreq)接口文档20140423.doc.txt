1．C＋＋接口


1.1 LJWF_Inits

Init the analyzer and prepare necessary data for LJWFaccording the
configure file.
int LJWF_Inits(const char *path, int encode, const char*sLicenceCode=0);

|Routine      |Required Header   |
|LJWF_Inits   |< LJWordFreq.h>   |


Return Value
Return non 0 if init succeed. Otherwise return lower 0.
Parameters
sDataPath: Initial Directory Path, where Data directory stored.  the
default value is 0, it indicates the initial directory is current working
directory path
int encoding: 默认为 GBK_CODE (GBK encoding), 也能是 UTF8_CODE (UTF8
encoding) 和 BIG5_CODE (BIG5 encoding).
char* sLicenceCode: license code, special use for some commercial users.
Other users ignore the argument
Remarks
The LJWF_Inits function must be invoked before any operation with
LJWordFreq. The whole system need call the function only once before
starting LJWordFreq. When stopping the system and make no more operation,
LJWF_Inits should be invoked to destroy all working buffer. Any operation
will fail if init do not succeed.
LJWF_Inits fails mainly because of two reasons: 1) Required data is
incompatible or missing 2) invalid parameters. Moreover, you could learn
more from the log file 2014****.log in the current directory.
Example
int main(int argc, char* argv[])
{
    if(LJWF_Inits("",0) < 0)
    {
       printf("初始化失败\n");
       return -1;
    }

    if(!LJWF_WordFreqOutput("E:\\LJParse\\演示语料\\", "./"))
    {
       printf("词频统计失败\n");
       return -1;
    }
    LJWF_Exits();
    return 0;
}

1.2 LJWF_Exits

Exit the program and free all resources and destroy all working buffer used
in LJWordFreq.
void LJWF_Exits();
|Routine      |Required Header   |
|LJWF_Exits   |< LJWordFreq.h>   |


Return Value
NULL.
Parameters
none
Remarks
The LJWF_Exits function must be invoked while stopping the system and make
no more operation. And call LJWF_Exits  function to restart LJWordFreq.
Example
int main(int argc, char* argv[])
{
    if(LJWF_Inits("",0) < 0)
    {
       printf("初始化失败\n");
       return -1;
    }

    if(!LJWF_WordFreqOutput("E:\\LJParse\\演示语料\\", "./"))
    {
       printf("词频统计失败\n");
       return -1;
    }
    LJWF_Exits();
    return 0;
}

1.3 LJWF_WordFreqOutput


bool LJWF_WordFreqOutput(const char* lpszFilename, char* DataPath);

|Routine            |Required Header     |
|LJWF_WordFreqOutput|< LJWordFreq.h>     |


Return Value
Return true if succeed,or return false.
Parameters
lpszFilename:  [IN] directory that process statistics of word frequency   or
not;
DataPath:   [OUT] directory that dictionary file [Data].
Remarks
The dynamic library depends on NLPIR.dll or  libNLPIR.so.and  you  will  get
the result from current directory ,the directory  “/output”  ,you  wil  find
three .xls file
[Bigrams.xls] :[FreqSortByWord.xls]:[FreqTrans.xls]
So you will find the answer from this file .
The   directory   [/seg]   include   all   file   have   been   process   by
NLPIR_FileProcess;
Example
int main(int argc, char* argv[])
{
    if(LJWF_Inits("",0) < 0)
    {
       printf("初始化失败\n");
       return -1;
    }

    if(!LJWF_WordFreqOutput("E:\\LJParse\\演示语料\\", "./"))
    {
       printf("词频统计失败\n");
       return -1;
    }
    LJWF_Exits();
    return 0;
}

3 常见问题（FAQ）


Q1: LJWordFreq组件初始化老是失败

Answer:
请您保证NLPIR.user文件未过期，另外可参考“20***.err”文件查看错误信息

Q2: java获取xls中词频统计结果失败

Answer:
请您保证java的LJWF_WordFreqOutput接口的第一个参数的路径不是中文路径，Java中S
tring默认是unicode编码格式，传入到该接口中可能无法访问该路径，另外可参考“20*
**.err”文件查看错误信息
